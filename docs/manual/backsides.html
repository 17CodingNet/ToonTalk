Backsides of widgets

You can open up the backside of a widget by clicking on it. If the widget is running you may need to stop it first. If the widget is not in a workspace (is on a web page directly) then clicking on it runs it so you need to hold donw the Alt key (or Command key on Macs) when clicking. Backsides are different for each kind of widget. They all have a > button to open up advanced settings (except nests which don't have any advanced settings). Widget specific settings are documented along with widget's documentation.

Note that the > button has become < and clicking it will hide the advanced options.

  • The input area preceded by I'm is where you can document the widget. What is entered will also appear as part of the tool tip for the widget.
  • The input area preceded by My name is is where you can give a nicer name to birds, nests, and robots. Note that for boxes the area is preceded by The labels of my holes are. The contents of the box's text area is used to put labels under each hole. The hole labels are seperated by semi-colons (;).

  • The Make a sensor nest button makes a sensor for this widget.
  • The Make a function bird makes a function bird for this kind of widget (or is disabled if there are none).
  • The Copy when dragged check box turns the widget into an infinite stack where dragging off copies leaves the stack unchanged.
  • The Drag the ... I'm on when dragging check box will only appear if the widget is contained in something else. This is particularly useful if many widgets are on a nest since it enables the dragging of the nest and its contents.
  • The Mouse click opens the backside only if I'm stopped check box is very useful for widgets with robots on the back that respond to mouse clicks. Enabling this means the backside won't appear when clicking a widget as part of your program's interface.
